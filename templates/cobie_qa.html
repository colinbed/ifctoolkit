COBie QA/QC guidance

  • Required fields: mandatory COBie cells that must be populated for useful exchange.
  • PickLists: project allowed values used to keep categories, units and status values consistent.
  • Cross-sheet references: links such as Component.TypeName to Type.Name and Space.FloorName to Floor.Name.
  • Type vs Component: Types describe asset classes; Components are individual maintainable assets.
  • ExternalIdentifier: the persistent source-system identifier used for traceability.
  • CreatedBy / CreatedOn: audit fields identifying who created records and when.

Limitation: The checker supports COBie validation and QA workflows, but it does not guarantee project, contractual, client, ISO 19650, security, or asset information compliance by itself. Project teams remain responsible for checking against their own AIR, EIR, BEP, exchange information requirements, security protocols and contractual obligations.
