IFC checking workflow

  1. Prepare the IFC export
  2. Upload the model
  3. Run available checks
  4. Review warnings and failures
  5. Export results
  6. Feed issues back to the authoring team
  7. Re-check after correction

Data extraction

IFC Toolkit can help extract model data into structured schedules or spreadsheets for element review, property review, COBie support, classification review, asset data checks and QA reporting.

Preparing good IFC files

  • Use agreed project naming standards.
  • Export correct IFC entities where possible.
  • Avoid unnecessary proxy objects.
  • Populate required properties before export.
  • Use consistent classification references.
  • Check spatial structure before issuing.
  • Coordinate IFC export settings across disciplines.
