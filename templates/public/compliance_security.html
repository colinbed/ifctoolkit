Compliance at the core of everything we do.

Construction project information can contain sensitive commercial, operational and security-relevant data. IFC Toolkit is designed to support information management workflows without creating unnecessary data retention or security risk.

{{ ui.cards([('Minimise Data','We only process the information required to complete the selected validation or transformation task.'),('Session-Based Processing','Uploaded files are handled through temporary sessions and can be purged when the session ends.'),('Secure by Design','The platform is being developed with secure configuration, access control and responsible data handling in mind.'),('Transparent Controls','We aim to make data processing, retention and security assumptions clear to users and organisations.')]) }}

Temporary processing by default.

  1. User uploads file
  2. Secure processing session created
  3. Validation or transformation runs
  4. Results are generated
  5. User downloads outputs
  6. Source files are purged according to retention settings

UK-hosted deployment model.

IFC Toolkit is deployed on UK-based Civo infrastructure for the current production environment. The platform is being structured so future enterprise deployments can support defined hosting, retention and security requirements.

Responsible AI usage.

IFC Toolkit may use AI-assisted features in future to help explain validation results, summarise issues or support QA workflows.

We do

  • Use AI to support clearer information review where appropriate.
  • Keep human review central to decision-making.
  • Aim to provide explainable outputs.

We do not

  • Use uploaded project files to train public AI models.
  • Sell project data.
  • Share uploaded project files for unrelated purposes.
